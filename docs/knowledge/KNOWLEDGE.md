# KNOWLEDGE

## decisions

- 問題ファイルの正規配置は `10刻み親フォルダ -> pNNN -> files` とした。
- 問題配下の `.md` はSQLソースとみなし、内容を変えず `.sql` に変換した。
- 同じ問題番号に属する複数画像・複数SQLは欠番扱いせず、元ファイル名を保持したまま同一 `pNNN` フォルダへまとめる方針とした。
- 問題文は PDF の一覧表から抽出し、各問題フォルダへ `pNNN.md` として格納する方針とした。
- 複数問題が1つにまとまっている SQL は、番号境界が判別できる限り、問題番号ごとの `pNNN.sql` に分割する方針とした。

## constraints

- `data` と `docs` は再編成対象外とした。
- Git リポジトリではないため、変更履歴の記録はドキュメント側に残した。
- PDF 抽出用の外部ライブラリ追加はネットワーク解決不可で失敗したため、ローカルに存在した `pdfplumber` を利用した。

## implementation

- 初期状態では `1-10`、`11-20`、`21-30` とそれ以降で構成が不統一だった。
- 再編成後は全親フォルダで `pNNN` フォルダを揃え、問題ファイルの親子関係を統一した。
- `41-50` に誤って残っていた `p051`-`p060.md` は `51-60` の `.sql` とハッシュ一致を確認して整理した。
- `docs/[Questions]100knocks.pdf` は表形式で抽出でき、100 問を欠番なく取得できた。
- `pNNN.md` には `P-xxx`、大分類、小分類、設問主旨、難易度、問題文を Markdown 形式で保存した。
- `41-50\p042-50.sql` は `42` から `50` の番号行で区切られていたため、その境界を使って `p042.sql` から `p050.sql` に分割した。
- 分割後は `41-50` 直下の SQL をなくし、すべて `p041` から `p050` フォルダ配下に揃えた。
