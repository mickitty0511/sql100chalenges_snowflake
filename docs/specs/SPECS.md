# SPECS

## Scope

この作業の対象は、SQL 100 本ノックの問題ファイル配置の整理である。`data` と `docs` は対象外とし、`1-10` から `91-100` までの問題フォルダ配下だけを再編成する。

## Structure Rules

- 各10刻み親フォルダ配下には、該当範囲の `pNNN` フォルダをすべて用意する。
- 問題に紐づく SQL は `pNNN` フォルダ直下に `.sql` で保存する。
- 問題に紐づく画像は `pNNN` フォルダ直下に保存する。
- 問題文は `docs/[Questions]100knocks.pdf` を元に、各 `pNNN` フォルダ直下へ `pNNN.md` として保存する。
- 同一問題番号に複数画像または複数SQLバリエーションがある場合は、元ファイル名を保持したまま同じ `pNNN` フォルダに共存させる。
- 問題ファイルの親フォルダは必ず問題番号と一致する10刻み範囲に置く。

## Conversion Rules

- 問題配下の `.md` は内容を変更せず `.sql` に変換する。
- 既に同内容の `.sql` がある場合、誤配置の `.md` は重複として整理する。
- 問題文 Markdown は PDF から抽出した大分類、小分類、設問主旨、難易度、問題文を含める。
- SQL 用途ではない `pNNN.md` は問題文ファイルとして保持する。
- 複数問題分をまとめた SQL は、問題番号ごとに単独の `pNNN.sql` に分割して各 `pNNN` フォルダへ保存する。

## Acceptance Criteria

- 各10刻み親フォルダ配下に `pNNN` フォルダが 10 件ずつある。
- 各 `pNNN` フォルダに対応する `pNNN.md` が存在する。
- `pNNN.md` の総数が 100 件である。
- 問題ごとの SQL は対応する `pNNN` フォルダ内に単独ファイルとして存在する。
- 問題番号と親10刻みフォルダの不整合ファイルが 0 件である。
- `p027`、`p028`、`p030`、`p040`、`p092` のような複数バリエーションを持つ問題で、元ファイル名が保持されている。
